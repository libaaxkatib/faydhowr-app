# Fayadhowr Project Constitution

## 1. Project Vision

Fayadhowr shall be developed as a reliable, maintainable, and user-focused project that fulfills its approved business purpose.

## 2. Project Goals

- Deliver the approved project scope.
- Maintain a high standard of quality, security, and usability.
- Ensure the system remains scalable, modular, and maintainable.

## 3. Business Rules Protection

- Never change approved business rules without approval.
- Never add features outside the approved scope.

## 4. UI/UX Protection

- Never redesign approved UI.
- Only implement approved designs.

## 5. Architecture Protection

- Follow the approved architecture.
- Keep the project modular and maintainable.

## 6. Code Quality

- Follow SOLID principles.
- Keep code clean and reusable.
- Avoid duplicate logic.

## 7. Database Rules

- Use migrations only.
- Never modify the existing schema without approval.

## 8. API Rules

- Use REST APIs only.
- Provide consistent JSON responses.
- Use proper HTTP status codes.

## 9. Security Rules

- Validate all input.
- Use authorization.
- Protect sensitive data.

## 10. Documentation Rules

- Every implementation must remain traceable.
- Documentation must stay synchronized with implementation.

## 11. AI Rules

- Never guess.
- Ask when requirements are unclear.
- Explain implementation before coding.

## 12. Final Rule

This Constitution overrides every other project document.
