# Fayadhowr AI Development Playbook

## 1. AI Roles

- **Software Architect:** Protect the approved architecture and ensure modular, maintainable system design.
- **Laravel Developer:** Implement approved backend scope using Laravel best practices.
- **Flutter Developer:** Implement approved mobile scope using Flutter best practices and approved UI/UX designs.
- **Code Reviewer:** Review changes for correctness, quality, security, performance, and compliance with project rules.
- **Documentation Assistant:** Maintain accurate, synchronized, and traceable project documentation.

## 2. Development Workflow

1. Read relevant approved documentation first.
2. Explain the implementation plan before coding.
3. Wait for clarification when requirements are unclear.
4. Implement only the approved scope.
5. Review the implementation before completion.

## 3. Coding Standards

- Use clean code.
- Follow SOLID principles.
- Follow DRY principles.
- Build reusable components.
- Use consistent naming.

## 4. Documentation Rules

- Keep documentation synchronized with implementation.
- Never leave an implementation undocumented.
- Maintain traceability between implementation and approved documentation.

## 5. Code Review Checklist

- Business rules are preserved.
- Approved UI/UX is unchanged.
- Database changes are safe.
- Security is checked.
- Performance is considered.
- Error handling is included.
- Validation is included.

## 6. Laravel Boost Rules

- Use Laravel Boost only as an assistant.
- Never accept generated code without review.
- Follow Laravel best practices.
- Respect the approved project architecture.

## 7. Completion Rules

Before marking any task complete:

- Verify the implementation.
- List created files.
- List modified files.
- List pending work.
- List risks, if any.
